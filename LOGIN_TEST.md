# 登录功能测试指南

## 🧪 测试步骤

### 1. 启动开发服务器

```bash
npm run dev
```

服务器会在 http://localhost:3000 启动（如果端口被占用，会使用 3001）

### 2. 配置环境变量

确保项目根目录有 `.env.local` 文件：

```bash
# 管理员账号配置
ADMIN_USERNAME=admin
ADMIN_PASSWORD=xhblogs2026
ADMIN_NAME=XiaoQuanJie
ADMIN_EMAIL=2023683967@qq.com

# NextAuth 配置
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your-secret-key-here
```

### 3. 访问登录页面

打开浏览器访问：http://localhost:3000/login

### 4. 测试登录

**输入账号信息：**
- 用户名：`admin`
- 密码：`xhblogs2026`

**点击"登录"按钮**

### 5. 验证登录成功

- ✅ 登录成功后应该自动跳转到 http://localhost:3000/publish
- ✅ 导航栏右侧应该显示用户头像和"Admin"标签
- ✅ 应该有"登出"按钮

### 6. 测试发布功能

- ✅ 访问 http://localhost:3000/publish 应该正常显示
- ✅ 可以发布文章、说说、杂谈

### 7. 测试登出

- ✅ 点击"登出"按钮
- ✅ 应该跳转到首页
- ✅ 导航栏应该显示"登录"按钮

### 8. 测试权限保护

- ✅ 未登录时访问 http://localhost:3000/publish 应该重定向到登录页面
- ✅ 未登录时调用 POST API 应该返回 401 错误

## 🔧 常见问题

### 问题 1：端口被占用

**现象**：服务器在 3001 端口启动

**解决**：
- 访问 http://localhost:3001 而不是 3000
- 或者关闭占用 3000 端口的程序

### 问题 2：登录失败

**现象**：输入正确的用户名和密码后仍然登录失败

**解决**：
1. 检查 `.env.local` 文件是否存在
2. 检查环境变量是否正确
3. 重启开发服务器

### 问题 3：API 返回 500 错误

**现象**：控制台显示 `TypeError: Function.prototype.apply was called on #<Object>`

**解决**：
1. 确保使用的是修复后的代码
2. 重启开发服务器
3. 清除浏览器缓存

### 问题 4：Middleware 弃用警告

**现象**：控制台显示 `The "middleware" file convention is deprecated`

**解决**：
- 这个警告可以忽略，功能正常工作
- 或者更新到最新的 Next.js 版本
