# XHBlogs 本地发布指南

## 📝 启动方式

```bash
# 方式 1：使用 npm 脚本
npm run publish

# 方式 2：直接运行
node scripts/local-publish.js
```

---

## 🎯 功能菜单

启动后会显示菜单：

```
🚀 XHBlogs 本地发布工具

请选择内容类型:
  1. 📝 文章 (posts)
  2. 💬 说说 (moments)
  3. 📖 杂谈 (chatters)
  0. 退出

请输入选项 (0-3):
```

---

## 1️⃣ 发布文章

选择 `1` 后，按提示输入：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 标题 | 文章标题 | ✅ 必填 |
| 描述 | 文章简介 | 可选 |
| 封面图链接 | 封面图片 URL | 可选 |
| 标签 | 用逗号分隔 | 可选 |
| 内容 | 输入 `END` 结束 | ✅ 必填 |

**示例交互：**
```
📝 发布文章

标题: 我的第一篇文章
描述: 这是一篇测试文章
封面图链接: https://example.com/cover.jpg
标签: 教程,前端,React

请输入内容 (输入 END 结束):
# 这是我的文章标题
这是文章内容...
END

✅ 文章发布成功！
   文件: /path/to/posts/我的第一篇文章.md
```

---

## 2️⃣ 发布说说

选择 `2` 后，按提示输入：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 内容 | 说说内容 | ✅ 必填 |
| 位置 | 当前位置 | 可选 |
| 图片链接 | 用逗号分隔多个 URL | 可选 |

**示例交互：**
```
💬 发布说说

内容: 今天天气真好！☀️
位置: 📍 北京
图片链接: https://example.com/1.jpg, https://example.com/2.jpg

✅ 说说发布成功！
   文件: /path/to/moments/moment-1719012345678.md
```

---

## 3️⃣ 发布杂谈

选择 `3` 后，按提示输入：

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 标题 | 杂谈标题 | ✅ 必填 |
| 心情 | 如 😊 开心 | 可选 |
| 封面图链接 | 封面图片 URL | 可选 |
| 标签 | 用逗号分隔 | 可选 |
| 内容 | 输入 `END` 结束 | ✅ 必填 |

**示例交互：**
```
📖 发布杂谈

标题: 学习笔记
心情: 🤔 思考
封面图链接:
标签: 学习,笔记

请输入内容 (输入 END 结束):
## 今天学到了什么
...
END

✅ 杂谈发布成功！
   文件: /path/to/chatters/学习笔记.md
```

---

## 4️⃣ 批量发布

发布完成后会询问：
```
继续发布？(Y/n):
- 按 `Y` 或 `Enter` 继续发布
- 按 `n` 退出
```

## 📂 生成的文件结构

### 文章文件 (`posts/xxx.md`)
```markdown
---
title: "文章标题"
date: "2026-06-21T15:30:00.000Z"
tags: ["标签1", "标签2"]
cover: "封面图链接"
description: "文章描述"
status: "published"
---

文章内容...
```

### 说说文件 (`moments/moment-xxx.md`)
```markdown
---
date: "2026-06-21T15:30:00.000Z"
location: "📍 位置"
images: ["图片1.jpg", "图片2.jpg"]
status: "published"
---

说说内容...
```

### 杂谈文件 (`chatters/xxx.md`)
```markdown
---
title: "杂谈标题"
date: "2026-06-21T15:30:00.000Z"
tags: ["标签1", "标签2"]
mood: "😊 开心"
cover: "封面图链接"
status: "published"
---

杂谈内容...
```

## 🚀 完整工作流程

```bash
# 1. 本地发布内容
npm run publish

# 2. 查看生成的文件
ls posts/ moments/ chatters/

# 3. 提交到 Git
git add .
git commit -m "feat: 添加新文章"

# 4. 推送到 GitHub
git push

# 5. Vercel 自动部署
# 访问 https://blog.jayvex.xyz 查看效果
```

---

## ⚠️ 注意事项

1. **内容格式**：支持 Markdown 语法
2. **图片链接**：建议使用图床服务（如 PicX、GitHub 等）
3. **标签格式**：多个标签用英文逗号分隔
4. **文件命名**：标题会自动转换为 URL 友好的 slug
5. **重复检测**：文章和杂谈会检测同名文件，说说使用时间戳 ID 不会重复

## 🔧 常见问题

### Q: 如何编辑已发布的内容？
A: 直接编辑对应的 `.md` 文件即可。

### Q: 如何删除内容？
A: 删除对应的 `.md` 文件，然后提交推送。

### Q: 如何设置草稿？
A: 修改生成文件中的 `status: "draft"`。

### Q: 内容支持哪些格式？
A: 支持完整的 Markdown 语法，包括标题、列表、代码块、图片、链接等。
