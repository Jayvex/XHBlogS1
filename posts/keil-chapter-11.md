## 引言

蜂鸣器是一种常见的声音输出设备，广泛应用于各种电子设备中。本章将学习：
- 蜂鸣器的工作原理和驱动方式
- 通过GPIO翻转产生方波信号
- 使用定时器中断产生不同频率的方波
- 实现音乐播放功能

## 项目列表

1. **11-1 蜂鸣器播放提示音**：学习蜂鸣器的基本驱动，实现按键控制蜂鸣器发声
2. **11-2 蜂鸣器播放音乐**：使用定时器中断产生不同频率，实现自动播放音乐

## 项目1：11-1 蜂鸣器播放提示音

### 项目目标
通过按键控制蜂鸣器播放提示音，按下按键后蜂鸣器发出约1KHz的提示音，同时数码管显示按键编号。

### 代码解析

**Buzzer.h（头文件）：**
```c
#ifndef __BUZZER_H__
#define __BUZZER_H__

void Buzzer_Time(unsigned int ms);
#endif
```

**Buzzer.c（驱动实现）：**
```c
#include <reg52.h>

// 蜂鸣器引脚定义
#define Buzzer P25

// 500微秒延时函数 @11.0592MHz
void Buzzer_Delay500us()
{
    unsigned char i;
    i = 227;
    while (--i);
}

// 蜂鸣器发声函数
void Buzzer_Time(unsigned int ms) {
    unsigned int i;
    for (i = 0; i < ms * 2; i++) {
        Buzzer = !Buzzer;       // 翻转蜂鸣器引脚电平
        Buzzer_Delay500us();    // 延时500微秒
    }
}
```

**main.c（主程序）：**
```c
/**
 * CPU: 89C52
 * Freq: 11.0592MHz
*/

#include <reg52.h>
#include "delay.h"
#include "key.h"
#include "nixie.h"
#include "Buzzer.h"

unsigned char KeyNum;

void main()
{
    nixie(1, 0);
    while (1)
    {
        KeyNum = key();
        if (KeyNum != 0) {
            Buzzer_Time(1000);  // 蜂鸣器响1000毫秒
            nixie(1, KeyNum);   // 数码管显示按键号
        }
    }
}
```

**代码解释：**
- `Buzzer_Delay500us()`：基于11.0592MHz晶振的500微秒延时
- `Buzzer_Time(ms)`：发声函数，通过翻转引脚产生方波
- 频率计算：每500us翻转一次，周期1ms，频率约1KHz

### 关键知识点
1. **方波产生**：通过快速翻转GPIO引脚电平产生方波信号
2. **频率控制**：延时时间决定方波频率，500us延时对应约1KHz
3. **发声时长**：通过循环次数控制发声持续时间
4. **有源vs无源蜂鸣器**：有源蜂鸣器给电即响，无源蜂鸣器需要方波驱动

### 遇到的问题和解决方法
**问题**：蜂鸣器没有声音。

**解决**：检查以下几点：
1. 确认蜂鸣器类型（有源/无源）
2. 检查引脚连接是否正确
3. 确认驱动电流是否足够（可能需要三极管驱动）
4. 检查延时函数是否准确

## 项目2：11-2 蜂鸣器播放音乐

### 项目目标
使用定时器中断产生不同频率的方波，配合乐谱数组实现自动播放音乐。

### 功能说明
- 使用定时器0中断产生精确的方波频率
- 通过频率表存储各音符对应的定时器初值
- 乐谱数组存储音符和时值信息
- 自动播放音乐，音符间有短暂停顿

### 代码解析

**timer0.h / timer0.c（定时器模块）：**
```c
#include <reg52.h>

// 定时器0初始化函数 @11.0592MHz
void timer0_init()
{
    TMOD &= 0xF0;      // 设置定时器模式
    TMOD |= 0x01;      // 设置定时器模式
    TL0 = 0x66;        // 设置定时初值
    TH0 = 0xFC;        // 设置定时初值
    TF0 = 0;           // 清除TF0标志
    TR0 = 1;           // 定时器0开始计时
    ET0 = 1;           // 使能定时器0中断
    EA = 1;            // 使能总中断
    PT0 = 0;           // 定时器0中断优先级为低
}
```

**main.c（主程序）：**
```c
/**
 * CPU: 89C52
 * Freq: 11.0592MHz
 * 歌曲: 偏爱
 * 调号: C调
 * 节拍: 4/4拍
*/

#include <reg52.h>
#include "delay.h"
#include "timer0.h"

#define Buzzer P25
#define Speed 500

// 音符与索引对应表
#define P   0       // 休止符
#define L1  1       // 低音1
#define L1_ 2       // 低音1#
#define M1  13      // 中音1
#define H1  25      // 高音1
// ... 其他音符定义

// 音符频率表（定时器初值）
unsigned int code FreqTable[] = {
    0,
    63628, 63731, 63835, 63928, 64021, 64103, 64185, 64260, 64331, 64400, 64463, 64528,
    64580, 64633, 64684, 64732, 64777, 64820, 64860, 64898, 64934, 64968, 65000, 65030,
    65058, 65085, 65110, 65134, 65157, 65178, 65198, 65217, 65235, 65252, 65268, 65283,
};

// 乐谱数组
unsigned char code Music[] = {
    // 音符, 时值
    H1, 3,
    M7, 2,
    M5, 1,
    H1, 1,
    H5, 1,
    // ... 更多音符
    0xff,  // 结束标志
};

unsigned char FreqSelect, MusicSelect;

void main()
{
    timer0_init();
    while (1)
    {
        if (Music[MusicSelect] != 0xff) {
            FreqSelect = Music[MusicSelect];
            MusicSelect++;
            delay(Speed / 4 * Music[MusicSelect]);  // 根据时值延时
            MusicSelect++;
            TR0 = 0;     // 停止定时器
            delay(5);    // 音符间短暂停顿
            TR0 = 1;     // 启动定时器
        } else {
            TR0 = 0;     // 播放结束
            while (1);
        }
    }
}

// 定时器0中断服务函数
void timer0_rountine() interrupt 1 {
    TL0 = FreqTable[FreqSelect] % 256;  // 设置定时初值
    TH0 = FreqTable[FreqSelect] / 256;  // 设置定时初值
    Buzzer = !Buzzer;                    // 翻转蜂鸣器
}
```

**代码解释：**
- `FreqTable[]`：存储各音符对应的定时器初值，初值决定中断频率
- `Music[]`：存储音符索引和时值对，0xff作为结束标志
- 定时器中断：每次中断翻转蜂鸣器，产生对应频率的方波
- 主循环：读取乐谱，设置频率，根据时值延时

### 关键知识点
1. **定时器中断**：使用定时器0产生精确的中断间隔
2. **频率表**：存储各音符对应的定时器初值
3. **音符表示**：使用宏定义表示低音(L)、中音(M)、高音(H)及升半音(_)
4. **时值控制**：时值单位为1/4拍，通过延时函数控制音符持续时间
5. **断奏效果**：音符间短暂停顿，产生断奏效果

### 遇到的问题和解决方法
**问题1**：音符频率不准确。

**解决**：重新计算定时器初值，公式为：`初值 = 65536 - (晶振频率 / 12 / 目标频率 / 2)`

**问题2**：音乐播放速度不对。

**解决**：调整Speed宏定义的值，值越大播放越慢。

## 总结

通过第11章的学习，我掌握了蜂鸣器的基本使用方法：

1. **GPIO驱动**：通过翻转GPIO引脚产生方波信号驱动蜂鸣器
2. **频率控制**：延时时间决定方波频率，从而控制音调
3. **定时器应用**：使用定时器中断产生精确频率的方波
4. **音乐播放**：通过乐谱数组和频率表实现自动播放音乐
5. **模块化设计**：将蜂鸣器驱动封装为独立模块

蜂鸣器虽然简单，但通过它可以学习到定时器中断的应用，这是单片机开发中非常重要的技能。掌握了定时器的使用，后续可以实现更多复杂的功能，如PWM控制、精确延时等。
